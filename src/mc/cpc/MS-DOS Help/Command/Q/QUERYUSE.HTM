QUERY USER

Description | Syntax | Parameters | Switches | Related | Notes | Examples | Errorlevels | Availability

Displays information about user sessions on a terminal server.

Syntax

QUERY USER /?

QUERY USER [{user_name | session_name | session_id}] [/server:server_name]

Parameters
session_id (NTXP)
Specifies the ID of the session you want to query.
If omitted, of all users who are logged on to the server is returned.
session_name (NTXP)
Specifies the name of the session you want to query.
If omitted, of all users who are logged on to the server is returned.
user_name (NTXP)
Specifies the logon name of the user you want to query.
If omitted, of all users who are logged on to the server is returned.
Switches
/? (NTXP)
Displays help.
/server:server_name (NTXP)
Specifies the terminal server you want to query. Otherwise, the current terminal server is used.
Required only if you use query user from a remote server.
Related

none.

Notes

Must have Full Control permission or Query Information special access permission.

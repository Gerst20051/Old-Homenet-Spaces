WINDOWS New Technology (NT) v3.5

Notes | Added | Changed | Removed | Files
  1. MINUMUM SYSTEM REQUIREMENTS
    1. CPU: 386 or 486/25MHz
    2. RAM: Workstation 12MB (Server 16MB)
    3. CD-ROM, 1.44 MB or 1.2 MB floppy or active network connection
    4. VIDEO: VGA level video support
    5. IDE, EIDE, SCSI or ESDI hard drive
    6. HARD DRIVE SPACE: 90MB
  2. First BETA Testing of Windows NT, 1992.
  3. Microsofts first true 32bit Operating System (code-name "Daytona"), September 6 1994.
  4. 9 million lines of code.
  5. Windows NT 3.5 Resource Kit, released December 16 1994.
  6. No known Service Packs.
  7. Windows NT system commands not found in MS-DOS
    1. ACLCONV Restores OS/2 HPFS386 ACLs to NTFS volumes.
    2. AT Schedules commands and programs to run on a computer at a specified time and date.
    3. CACLS Displays or modifies access control lists (ACLs) of files.
    4. CONVERT Converts file systems from FAT or HPFS to NTFS.
    5. DISKPERF Starts, stops, and displays system disk performance counter use.
    6. DOSONLY Prevents starting applications other than MS-DOS-based applications from the COMMAND.COM prompt.
    7. ECHOCONF Displays messages when reading the MS-DOS subsystem CONFIG.NT file.
    8. ENDLOCAL Ends localization of environment variables.
    9. FINDSTR Searches for text in files using regular expressions.
    10. NTCMDPROMPT Runs the Windows NT command interpreter, CMD.EXE, rather than COMMAND.COM after running a TSR or after starting the command prompt from within an MS-DOS application.
    11. POPD Changes to the directory last set with the pushd command.
    12. PUSHD Saves the current directory for use by the popd command, and then changes to the specified directory.
    13. SETLOCAL Begins localization of environmental variables.
    14. START Runs a specified program or command in a secondary window and in its own memory space.
    15. TITLE Sets the title of the command prompt window.
    16. && Command following this symbol runs only if the command preceding the symbol succeeds.
    17. || Command following this symbol runs only if the command preceding the symbol fails.
    18. & Separates multiple commands on the command line.
    19. ( ) Groups commands.
    20. ^ Escape character. Allows input of command symbols as text.
    21. ; or , Separates parameters.
  8. Changes to MS-DOS Commands
    1. CHCP Changes code pages for full-screen mode only.
    2. CMD CMD.EXE replaces COMMAND.COM.
    3. DEL New switches provide many more functions.
    4. DIR New switches provide many more functions.
    5. DISKCOMP Switches /1 and /8 are not supported.
    6. DISKCOPY Switch /1 is not supported.
    7. DOSKEY Available for all character-based programs that accept buffered input. Several other enhancements improve doskey.
    8. FORMAT 20.8 MB floptical drive supported. Switches /B, /S, and /U are not supported.
    9. KEYB KEYBOARD.SYS is no longer used.
    10. LABEL The symbols ^ and & can be used in a volume label.
    11. MODE Extensive changes.
    12. MORE New switches provide many more functions.
    13. PATH The %PATH% environment variable appends the current path to a new setting at the command prompt.
    14. PRINT Switches /B:size, /C, /M:ticks2, /P, /Q:qsize, /S:ticks3, /T, and /U:ticks1 are not supported.
    15. PROMPT New character combinations enable you to add ampersands ($a), parentheses ($c and $f), and spaces ($s) to your prompt.
    16. RECOVER Recovers files only.
    17. RMDIR New /S switch deletes directories containing files and subdirectories.
    18. SORT Does not require %TEMP% environment variable. File size is unlimited.
    19. XCOPY New switches provide many more functions.
  9. Unavailable MS-DOS Commands
    1. ASSIGN Not supported in Windows NT.
    2. BUFFERS Not currently supported.
    3. CHOICE Not currently supported.
    4. CTTY Not currently supported.
    5. DBLSPACE The Dblspace program is not supported.
    6. DEFRAG Windows NT automatically optimizes disk use.
    7. DELTREE The RMDIR /S command deletes directories containing files and subdirectories.
    8. DOSSHELL Unnecessary with Windows NT.
    9. DRIVEPARM Not currently supported.
    10. DRVSPACE The Drvspace program is not currently supported.
    11. EMM386 Unnecessary with Windows NT.
    12. FASTHELP This MS-DOS 6.0 command is the same as the Windows NT command help. Windows NT also provides an online Command Reference in the Windows NT Help icon in the Main program group.
    13. FDISK Disk Administrator prepares hard disks for use with Windows NT.
    14. INCLUDE Multi-configurations of the MS-DOS subsystem are not supported.
    15. INTERLNK The Interlnk program is not supported.
    16. INTERSVR The Intersvr program is not supported.
    17. JOIN Increased partition size and an improved file system eliminate the need to join drives.
    18. MEMMAKER Windows NT automatically optimizes the MS-DOS subsystems memory use.
    19. MENUCOLOR Multi-configurations of the MS-DOS subsystem are not supported.
    20. MENUDEFAULT Multi-configurations of the MS-DOS subsystem are not supported.
    21. MENUITEM Multi-configurations of the MS-DOS subsystem are not supported.
    22. MIRROR Not supported in Windows NT.
    23. MSAV The Msav program is not supported.
    24. MSBACKUP Windows NT provides the Backup utility (in the Administrativce Tool group) for computers with tape drives, or the backup and xcopy commands for computers without tape drives.
    25. MSCDEX It is unnecessary to configure the MS-DOS subsystem to use a CD-ROM drives. Windows NT provides access to CD-ROM drives for the MS-DOS subsystem.
    26. MSD Use the Windows NT Diagnostics program in the Administrative Tools group.
    27. NUMLOCK The numlock command is not currently supported.
    28. POWER The Power utility is not supported.
    29. SCANDISK The Scandisk utility is not supported.
    30. SMARTDRV Windows NT automatically provides caching for the MS-DOS subsystem.
    31. SUBMENU Multi-configurations of the MS-DOS subsystem are not supported.
    32. SYS Windows NT will not fit on a standard 1.2 MB or 1.44 MB floppy disk.
    33. UNDELETE Not supported in Windows NT.
    34. UNFORMAT Not supported in Windows NT.
    35. VSAFE The Vsafe program is not supported.
  10. New or Changed Commands from LAN Manager version 2.2
    1. AT In addition to local scheduling, you can remotely schedule events on a computer.
    2. IPXROUTE New command supports routing for the NWLink protocol on a token-ring network.
    3. NET ACCOUNTS Server roles cannot be set. Windows NT security controls lockout.
    4. NET COMPUTER This new command allows you to add or delete computers from a domain database. This command is available only on computers running Windows NT Server.
    5. NET CONFIG Peer functionality is inherent to Windows NT. Separate commands, such as net config peer are no longer required.
    6. NET CONFIG SERVER Most network services are self-configuring. The switches /AUTODISCONNECT:time, /SRVCOMMENT:"text", and /HIDDEN:{YES | NO} can be configured. The switch /srvhidden is now /HIDDEN:{YES | NO}.
    7. NET CONFIG WORKSTATION Most network services are self-configuring. The switches /CHARCOUNT:bytes, /CHARTIME:msec, and /CHARWAIT:sec can be configured.
    8. NET CONTINUE Use Print Manager to control printing. See NET PAUSE for the list of pausable services that can be continued.
    9. NET GROUP Net group manages global groups and is only for computers that are members of a domain.
    10. NET LOCALGROUP This new command manages local groups.
    11. NET PAUSE You can pause these network services: file server for macintosh and remoteboot (Windows NT Server only), ftp server, lpdsvc, net logon, network dde, network dde dsdm, nt lm security support provider, remote access server, schedule, server, simple tcp/ip services, or workstation.Use Print Manager to manage printers.
    12. NET PRINT Use Print Manager to manage printers.
    13. NET SEND Sending files is not supported. The /broadcast switch is not supported.
    14. NET SHARE Remote administration is automatic. Use Print Manager to share printers. Comm queues are not supported in this release.
    15. NET START You can start these network services: alerter, client service for netware, clipbook server, computer browser, dhcp client, directory replicator, eventlog, ftp server, lpdsvc, messenger, net logon, network dde, network dde dsdm, network monitoring agent, nt lm security support provider, ole, remote access connection manager, remote access isnsap service, remote access server, remote procedure call (rpc) locator, remote procedure call (rpc) service, schedule, server, simple tcp/ip services, snmp, spooler, tcp/ip netbios helper, ups, and workstation. These services are available only on Windows NT Server: file server for macintosh, gateway service for netware, microsoft dhcp server, print server for macintosh, remoteboot, windows internet name service. Note that services can be configured to start automatically.
      1. NET START ALERTER Self-configuring.
      2. NET START EVENTLOG This new service logs any significant system, security, and application occurrences that require users to be notified.
      3. NET START MESSENGER Self-configuring.
      4. NET START NET LOGON Self-configuring.
      5. NET START DIRECTORY REPLICATOR Use Server Manager to configure the Replicator service.
      6. NET START SERVER Use Server Manager to configure the Server service.
      7. NET START SNMP New options permit logging.
      8. NET START WORKSTATION The Workstation is configured at setup and in various applications.
    16. NET STATISTICS Peer functionality is an inherent part of Windows NT and no longer requires separate commands. The statistics log cannot be cleared.
    17. NET STOP You can stop all network services except eventlog. See NET START for a list of services that can be started and stopped.
    18. NET USE The /PERSISTENT switch has only yes and no values. Comm queues are not supported in this release. You can use NetWare volumes.
    19. NET USER Switches /logonserver, /maxstorage, /operator, and /privilege are not supported.
    20. NET VIEW New switch /DOMAIN permits viewing of domains and viewing of computers in a specified domain. You can view servers on NetWare networks.
  11. Superseded or Obsolete Commands from LAN Manager
    1. NET Use the Windows NT administrative tools to administer the network.
    2. NET ADMIN Use the Windows NT administrative tools to administer the network.
    3. NET ACCESS Use File Manager to set permissions on files.
    4. NET AUDIT Use the Event Viewer to keep track of network resource use.
    5. NET COMM Communication-device queues are not currently supported.
    6. NET CONFIG PEER Not needed with Windows NT.
    7. NET CONSOLE Use Windows NT security features to prevent unwanted access.
    8. NET COPY Use File Manager or the system copy command to copy files.
    9. NET DEVICE Use Print Manager to display information about shared printers.
    10. NET ERROR Use the Event Viewer to keep track of network resource use.
    11. NET FORWARD You cannot forward network messages with Windows NT.
    12. NET LOG Message logging is not supported.
    13. NET LOGOFF Logging off is an inherent part of Windows NT.
    14. NET LOGON Logging on is an inherent part of Windows NT.
    15. NET MOVE Use File Manager or the system copy and delete commands to move files.
    16. NET PASSWORD Press CTRL+ALT+DEL to change your password.
    17. NET RUN Programs cannot be executed remotely in Windows NT.
    18. NET SEPARATOR Use Print Manager to control separator pages.
    19. NET START NETPOPUP Windows NT processes network messages automatically.
    20. NET START NETRUN This command is not supported in Windows NT.
    21. NET START NVALERT The Nvalert service is not currently supported.
    22. NET START PEER Peer functionality is an inherent part of Windows NT.
    23. NET START TIMESOURCE The functionality is included in the Server service.
    24. NET STATUS The information can be found using net config server and net share.
    25. NET VERSION Use Program Managers Help About menu item to determine the current version you are using.
    26. NET WHO Use Server Manager to view users logged on to a server.

DOS v4.00

Notes | Added | Changed | Removed | Files
  • Europe only (SIEMENS?) multitasking (eDOS), withdrawn from market, 1986
  • Microsoft release (June 1988) added graphical/mouse interface.
  • IBM release (July 1988) added shell menu interface and larger hard disk partitions.
  • Added support for EMS (expanded memory) to several DOS features.
  • Supports larger hard disks (up to 2Gb), individual partitions can contain up to 512Mb.
  • Lack of compatibility with previous DOS versions and large RAM (about 80K) requirements result in v4.01.
  • DOS v3.3 to v4.0 upgrade solely done by IBM, later licenced back to Microsoft (large amounts of memory worked only with either of two expansion boards manufactured by IBM).
  • Commands introduced:
    1. DOSSHELL, graphical, interactive user interface.
    2. MEM provides summary of memory usage, including EMS.
    3. INSTALL lets you load TSRs in your Config.Sys file.
    4. SWITCHES
    5. TRUENAME (undocumented).
  • Commands enhanced:
    1. ANSI.SYS
    2. APPEND can locate or ignore .EXE, .COM. and .BAT files.
    3. BACKUP
    4. BUFFERS, EMS support.
    5. CHKDSK summarizes allocation unit usage.
    6. COUNTRY
    7. DEBUG, EMS support.
    8. DIR
    9. DISKCOMP
    10. DISKCOPY
    11. DISPLAY.SYS
    12. ERASE /P lets you OK each deletion of a wildcard filespec.
    13. FASTOPEN, EMS support.
    14. FDISK lets you create partitions larger than 32 Mb.
    15. FORMAT automatically prompts for a volume label. The DOS v4.01 boot sector contains a volume label and a volume serial number.
    16. GRAFTABL
    17. GRAPHICS lets you define a custom Printer Profile file.
    18. KEYB
    19. LABEL
    20. MODE is significantly enhanced:
      1. 43-line and 50-line screen modes are supported.
      2. Sets keyboard repeat rate.
      3. Displays device status.
      4. Uses intuitive, non-positional parameters.
    21. PRINTER.SYS
    22. REM
    23. REPLACE lets you overwrite read-only files on the target.
    24. SELECT became a no-holds-barred DOS installation program.
    25. SHARE is installed automatically to support large partitions.
    26. SYS is more powerful. It will move directory entries if needed and the two hidden system files need not be contiguous. Writes new style boot sector.
    27. TIME
    28. TREE
    29. VDISK.SYS (RAMDRIVE.SYS), EMS support.
    30. VOL
